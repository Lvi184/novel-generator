## 项目简介

### 什么是小说仿写系统？

小说仿写系统是一个基于 AI 的智能小说创作工具，它可以：

1. **深度解剖** 原著小说，提取故事骨架
2. **骨架变形** 将原著骨架映射到新的世界观设定
3. **逐章生成** 基于新骨架生成全新小说

### 核心设计理念

```
仿写 = 保留「骨+魂」，替换「皮+肉」

┌─────────────────────────────────────────────────┐
│  表层（皮）  → 文字/文风/修辞/语感               │  ← 可以换
│  中层（肉）  → 角色/设定/世界观/具体情节          │  ← 可以换
│  深层（骨）  → 故事结构/节奏曲线/冲突模式/情感弧线  │  ← 要保留
│  核层（魂）  → 主题/价值观/读者爽点设计            │  ← 要保留
└─────────────────────────────────────────────────┘
```

- **最近测试：** 2026-03-14 - 《我在精神病院学斩神》前5章完整测试

## 快速开始

```bash
cd case1
```

将你的小说 TXT 文件放到：
```
data/zhu_xian/input/novel.txt
```

- 编码：UTF-8
- 章节标题格式：`第一章 标题内容` 或 `第1章 标题内容`

```bash
# 分析前 5 章（快速测试）

python scripts/test_phase1_full.py
```

#### 4. 运行 Phase 2 - 骨架变形

```bash
python scripts/test_phase2.py
```

#### 5. 运行 Phase 3 - 逐章生成

```bash
# 生成第 1 章
python scripts/test_phase3.py

# 生成多章
python scripts/generate_chapters.py
```

生成的小说在：

## 详细使用指南

### 第一阶段：深度解剖（Phase 1）

#### 功能说明

从原著小说中提取结构化骨架，包括：
- 章节分割
- 逐章摘要
- 角色分析
- 递归摘要金字塔
- 全局故事骨架

#### 使用方法

**基础分析（前 N 章）：**

```bash
python scripts/test_phase1.py
```

编辑 `scripts/test_phase1.py` 修改参数：
```python
novel_path = "data/duo_qing_jian_ke_wu_qing_jian/input/novel.txt"
output_dir = "data/duo_qing_jian_ke_wu_qing_jian/analysis"
max_chapters = 5  # 只分析前 5 章
```

**完整分析（含递归摘要）：**

```bash
python scripts/test_phase1_full.py
```

#### 输出文件

```
data/your_novel/analysis_full/
├── chapter_summaries.json          # 逐章摘要
├── recursive_summary.json           # 递归摘要金字塔
├── global_analysis.json             # 全局分析
├── novel_skeleton.json              # 故事骨架
└── characters/                      # 角色档案
    ├── character_1.json
    └── ...
```

#### 递归摘要金字塔说明

```
Level 1 (原始) → Level 2 (组) → Level 3 (大组) → Full (全书)
   ↑                  ↑                ↑               ↑
每章摘要         每5-10章         每2-3组         综合摘要
```

完美解决长篇小说上下文窗口限制问题！

---

### 第二阶段：骨架变形（Phase 2）

#### 功能说明

将原著骨架映射到新的世界观设定：
- 世界观映射
- 角色映射
- 剧情节拍映射
- 章节大纲展开

#### 使用方法

编辑 `scripts/test_phase2.py` 配置新设定：

```python
# 新世界设定
new_world_setting = {
    "genre": "都市重生",
    "time_period": "2010年",
    "location": "深圳",
    "background": "主角林风，28岁程序员，重生回到2010年",
    "macguffin": "iPhone 15 Pro Max 里的未来知识（2010-2035）"
}

# 角色映射
character_mapping = {
    "李寻欢": {"name": "林风", "role": "主角", ...},
    "阿飞": {"name": "张磊", "role": "兄弟", ...},
    ...
}
```

运行变形：

```bash
python scripts/test_phase2.py
```

#### 输出文件

```
data/your_novel/transformed/
├── world_mapping.json               # 世界观映射
├── character_mapping.json           # 角色映射
├── beat_mapping.json                # 剧情节拍映射
├── chapter_outlines.json            # 章节大纲
└── transformed_outline.json         # 完整新大纲
```

### 第三阶段：逐章生成（Phase 3）

#### 功能说明

基于新大纲逐章生成小说，包含：
- 记忆检索（角色状态/伏笔/前情）
- 质量检查（1-10分评分）
- 自动重试（最多 2 次）
- 状态追踪

#### 使用方法

**生成单章：**

```bash
python scripts/test_phase3.py
```

**生成多章：**

```bash
python scripts/generate_chapters.py
```

**增强版生成（含质量检查）：**

```bash
python scripts/generate_enhanced.py
```

#### 配置生成参数

编辑生成脚本：

```python
config = {
    "chapter_num": 1,                    # 章节号
    "quality_threshold": 7,              # 质量阈值（1-10）
    "max_retries": 2,                    # 最大重试次数
    "style": "古龙风格",                  # 文风要求
    "temperature": 0.7,                  # 温度参数
    "presence_penalty": 0.1,             # 存在惩罚
    "frequency_penalty": 0.1             # 频率惩罚
}
```

#### 输出文件

```
data/your_novel/output_enhanced/
├── chapter_1.txt                     # 第 1 章
├── chapter_2.txt                     # 第 2 章
├── ...
├── quality_reports/                  # 质量报告
│   ├── chapter_1_quality.json
│   └── ...
└── state/                            # 状态追踪
    ├── chapter_1_state.json
    └── ...
```

## 常见问题

### Q1: 如何更换 LLM 模型？

**A:** 编辑配置文件：

```yaml
# config/config.yaml
model:
  interface: "OpenAI"
  base_url: "https://api.deepseek.com/v1"  # 更换 API 地址
  name: "deepseek-chat"                      # 更换模型名称
  api_key: "${DEEPSEEK_API_KEY}"             # 更换 API Key
```

支持的模型：
- 火山引擎豆包系列
- DeepSeek 系列
- OpenAI GPT 系列
- 任何兼容 OpenAI 格式的 API

---

### Q2: 如何避免生成重复内容？

**A:** 调整生成参数：

```python
config = {
    "temperature": 0.8,           # 增加随机性
    "presence_penalty": 0.2,      # 增加存在惩罚
    "frequency_penalty": 0.2,     # 增加频率惩罚
}
```

---

### Q3: 分析长篇小说需要很长时间，怎么办？

**A:** 有几个方案：

1. **只分析前 N 章：**
```python
max_chapters = 20  # 只分析前 20 章
```

2. **使用缓存：**
   - 系统会自动缓存 LLM 调用
   - 重复运行时会使用缓存，节省时间和成本

3. **分批处理：**
   - 先分析前 10 章
   - 再分析接下来的 10 章
   - 最后合并结果

---

### Q4: 生成的小说质量不高，怎么办？

**A:** 尝试以下方法：

1. **提高质量阈值：**
```python
quality_threshold = 8  # 从 7 提高到 8
```

2. **调整文风提示：**
```python
style = "严格古龙风格，短句，高对话占比，章末悬念"
```

3. **手动编辑中间结果：**
   - 编辑 `transformed_outline.json` 调整大纲
   - 编辑 `chapter_outlines.json` 调整章节细纲

4. **增加重试次数：**
```python
max_retries = 3  # 从 2 增加到 3
```

---

### Q5: 如何自定义 Prompt 模板？

**A:** 使用 `prompts.py` 工具：

```python
from prompts import PromptManager

# 加载模板
prompt_manager = PromptManager()
template = prompt_manager.get_template("chapter_generation")

# 修改模板
template = template.replace("古龙风格", "金庸风格")

# 保存模板
prompt_manager.save_template("chapter_generation_jin_yong", template)
```

或者直接编辑 `prompts/` 目录下的模板文件。

## 高级功能

### 1. 状态追踪器详解

**功能：** 完整追踪小说中的各种状态

**状态类型：**

```python
# 角色状态
character_state = {
    "location": "深圳南山区",          # 当前位置
    "emotion": "平静",                  # 当前情绪
    "health": "健康",                   # 健康状况

# 关系状态

    "林风-张磊": {
        "trust": 0.9,                   # 信任度（0-1）

    }
}

# 伏笔追踪

    }

```

### 2. 记忆检索系统

**功能：** 生成当前章节时自动检索相关记忆

**检索内容：**

1. **角色状态：** 当前各角色的状态
2. **伏笔：** 已埋设但未回收的伏笔
3. **前情摘要：** 前 3-5 章的摘要
4. **关系状态：** 角色间的关系状态
5. **已知信息：** 各角色知道什么

**Prompt 组装：**

```
┌─────────────────────────────────────────┐
│  固定区                                  │
│  - 世界观设定                            │
│  - 全书大纲                              │
│  - 角色卡片                              │
├─────────────────────────────────────────┤
│  动态区（记忆检索）                      │
│  - 前一章内容                            │
│  - 前 3-5 章摘要                        │
│  - 本章细纲                              │
│  - 相关记忆                              │
│  - 活跃伏笔                              │
├─────────────────────────────────────────┤
│  约束区                                  │
│  - 本章应避免                            │
│  - 连续性提醒                            │
└─────────────────────────────────────────┘
```

---

### 3. 质量检查系统

**检查维度：**

| 维度 | 说明 | 权重 |
|------|------|------|
| 文风一致性 | 是否符合指定文风 | 20% |
| 情节连贯性 | 与前文是否连贯 | 20% |
| 角色一致性 | 角色行为是否符合设定 | 20% |
| 悬念设置 | 章末悬念是否吸引人 | 15% |
| 文笔质量 | 文字是否流畅优美 | 15% |
| 伏笔处理 | 伏笔埋设/回收是否合理 | 10% |

**评分流程：**

1. 生成章节
2. 多维度质量检查
3. 计算总分（1-10分）
4. 如果分数 < 阈值，自动重试（带上次问题和建议）
5. 最多重试 2 次

---

### 4. 缓存系统

**功能：** 避免重复调用 LLM，节省成本和时间

**缓存机制：**

```
- 缓存键：SHA256(prompt + model + temperature)
- 存储：按日期分层存储（data/cache/YYYY-MM-DD/）
- 过期：默认 7 天自动过期
- 命中率：实时统计
```

**查看缓存统计：**

```bash
python scripts/llm_cache.py stats
```

**输出示例：**

```
=== 缓存统计 ===
总调用次数：156
命中次数：89
命中率：57.1%
节省成本：约 ¥2.30
```
